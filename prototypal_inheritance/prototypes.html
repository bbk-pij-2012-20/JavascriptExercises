prototypes

Every Javacript object has an internal property which is its protoype-object.
If a property is requested from an object (e.g. using a getter), the property
will be returned from that object. But if the object does not have that property
then the property will be looked for in the object's prototype object. If it is
not in the prototype object, then it will be looked for in the prototype object's
prototype, and so on.

On the other hand, if you assign a value to a property, that value will only be assigned
to the object and not to its prototype.

In other words: Sets are shallow, gets are deep.

var Bike = function() {

       console.log("..called Bike constructor..");

}

Bike.prototype.km = 0;

Bike.prototype.pedal = function(km_travelled) {

       console.log("..you're cycling.. ");
       this.km = this.km + km_travelled;  // 'this.km' on the right is taken from the prototype, 'this.km' on the left is set in the object.

}

var bike1 = new Bike();
var bike2 = new Bike();
bike1.pedal(20);  // sets (shallow) km in bike1 object only
console.log(bike1.km);  // gives 20
console.log(bike2.km);  // gives 0
