Example#1: hoisting

w3schools describes hoisting in terms of variable declarations that it says can be made after a variable has been used.

However, I find that in Venkat's talk, his focus on hoisting is from the perspective of functions that share the same name - as I have mimicked below
- and I find this striking because Javascript's behaviour is rather more counter-intuitive and therefore more pertinent for learning (how to not screw up in Javascript!)

Specifically, if you use a variable without declaring (i.e. preceeded with var), and then assigning a different value to the same variable name after using the variable,
Javascript still uses the value assigned earlier. This is not the case with functions, e.g. the function called nutella - defined on line 1 - is used on line 4, nutella is
then redefined later (on line 5), before being used again (on line 8). It's striking then that both function calls (from lines 4 and 8) are to the latter function definition (on line 5).



1:              function nutella(n){

2:                        console.log("Nutella has nuts numbering " + n);

3:              };

4:              nutella(10);   // not as expected, gives the console.log() output of the 2nd function definition (below)



5:              function nutella(n){

6:                        console.log("...but Nutella now has nuts numbering " + n);

7:              };

8:              nutella(20);   // as expected, gives the console.log() output of the 2nd function definition

Example#2: avoiding hoisting by using the preferred syntax of declaring the functions

var nutella = function(n){

          console.log("Nutella has nuts numbering " + n);

};

nutella(10);    // as expected, gives the console.log() output of the 1st function definition (above)

nutella = function(n){

          console.log("...but Nutella now has nuts numbering " + n);

};

nutella(10);   // as expected, gives the console.log() output of the 2nd function definition
